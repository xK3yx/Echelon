# Echelon v2.2

AI-assisted career intelligence tool. Describe your skills, personality, and interests; Echelon ranks career paths against your profile using a deterministic rule engine followed by a Groq LLM re-ranker, then generates per-career skill gap analyses, personalised learning roadmaps, and course recommendations. When no strong match is found, a Stage 2.5 LLM proposal step generates speculative career suggestions from your unique profile.

Built as a final-year diploma portfolio project.

## What's new in v2.2

| Feature | Details |
|---|---|
| **Resume import** | Upload a PDF, DOCX, or TXT resume to pre-fill skills, interests, and education. Multi-stage validation: size → MIME → text extraction → heuristic → LLM → confidence threshold. |
| **Dark / light mode** | System-default with manual toggle. Persists via `next-themes`. |
| **Course recommendations** | "Bridge the gap" section inside each career card. Searches YouTube and/or Tavily in parallel, LLM-ranks results, caches per career slug for 7 days. Degrades gracefully when API keys are absent. |
| **Public share links** | One-click share button copies a `/r/{id}` URL to clipboard. Public page renders the full analysis without authentication. |
| **PDF export** | Download button generates a print-ready A4 PDF via WeasyPrint. Includes score breakdown, skill gaps, and learning roadmap. |
| **Score breakdown** | "How we scored this" section in each career card: four weighted bars (skill 45%, optional skills 15%, personality 25%, interests 15%) plus total rule score and LLM confidence with inline explanations. |
| **Onboarding tour** | First-visit 4-step guided tour (react-joyride). Dismissed on completion or skip; flag stored in `localStorage`. |

## Honest disclaimers

- **Portfolio project** — academic portfolio piece, not a production product. Not validated by career counsellors or industry research.
- **Curated dataset** — the career database covers ~200 O\*NET occupations plus hand-authored entries. It does not cover all career options.
- **LLM non-determinism** — rankings, reasoning, and roadmaps are generated by the Groq API (`llama-3.3-70b-versatile`). Identical inputs may produce different outputs. "Re-run" intentionally exploits this.
- **Resume parsing accuracy** — extraction uses heuristics + LLM. Accuracy varies by resume format and layout. Always review pre-filled fields before submitting.
- **Confidence scores are approximations** — the rule-based score is a weighted sum of categorical matches. The LLM confidence is a model self-assessment. Neither is a statistically validated predictor of career success.
- **No authentication** — profile submissions are anonymous. Do not submit real personal data.
- **No partnerships** — Echelon has no relationships with YouTube, Tavily, course creators, employers, or educational institutions.
- **Not career advice** — output is illustrative. Consult a qualified career advisor before making major decisions.

## Architecture

```
Browser (Next.js 14 App Router)
    │
    ├─ /profile          → multi-step form, resume upload, onboarding tour
    ├─ /results/{id}     → career cards, course recommendations, share/PDF buttons
    └─ /r/{id}           → public share page (no auth required)
    │
    │  POST /api/profiles
    │  POST /api/recommendations
    │  GET  /api/recommendations/{id}
    │  POST /api/recommendations/{id}/share
    │  GET  /api/recommendations/{id}/public
    │  GET  /api/recommendations/{id}/pdf
    │  POST /api/resume/parse
    │  GET  /api/courses/recommend
    ▼
FastAPI (Python 3.11)
    │
    ├─ Stage 1: Rule scoring (deterministic)
    │    skill × 0.45 + optional × 0.15 + personality × 0.25 + interest × 0.15
    │    Education soft penalty: high_school + high-difficulty career → score × 0.7
    │    Top 10 candidates passed to Stage 2
    │
    ├─ Stage 2: Groq re-rank (llama-3.3-70b-versatile, temp 0.3)
    │    Returns top 5 with fit_reasoning, strengths, risks, confidence
    │
    ├─ Stage 2.5: LLM career proposal (only when top score < propose_threshold)
    │    Generates speculative careers not in the database
    │
    ├─ Stage 3: Batched gap tagging + roadmap
    │    3-phase roadmap per career; skill difficulty tagged by llama-3.1-8b-instant
    │
    ├─ Resume parsing: size → MIME → text extraction → heuristic → LLM → threshold
    │
    └─ Course recommendations: YouTube + Tavily → LLM ranking → 7-day cache
    │
    PostgreSQL (SQLAlchemy async + Alembic migrations)
```

## Required environment variables

| Variable | Required | Description |
|---|---|---|
| `DATABASE_URL` | Yes | PostgreSQL connection string (`postgresql+asyncpg://...`) |
| `GROQ_API_KEY` | Yes | Groq API key — [console.groq.com](https://console.groq.com) |
| `ADMIN_TOKEN` | Recommended | Secret token for admin endpoints. Empty = admin disabled. |
| `NEXT_PUBLIC_API_URL` | Yes (frontend) | Backend API base URL, e.g. `http://localhost:8000/api` |
| `PUBLIC_BASE_URL` | Production | Frontend origin for CORS + share links, e.g. `https://echelon.vercel.app` |
| `YOUTUBE_API_KEY` | Optional | YouTube Data API v3. Course section hidden if absent. |
| `TAVILY_API_KEY` | Optional | Tavily Search API. Falls back to YouTube-only if absent. |
| `MAX_UPLOAD_SIZE_MB` | Optional | Max resume file size in MB (default `5`) |
| `RESUME_CONFIDENCE_THRESHOLD` | Optional | Combined parse confidence floor (default `0.4`) |
| `PROPOSE_THRESHOLD` | Optional | Rule score below which LLM career proposal triggers (default `0.4`) |
| `COURSE_CACHE_TTL_DAYS` | Optional | Days to cache course results per career slug (default `7`) |

Copy `.env.example` to `.env` and fill in the required values.

## Running locally

### Prerequisites

- Docker Desktop (or a local Postgres instance)
- Node.js 18+

### Quick start

```bash
# 1. Clone and configure
cp .env.example .env
# Edit .env — set DATABASE_URL and GROQ_API_KEY at minimum

# 2. Start backend + database
docker compose up --build -d

# 3. Run migrations and seed
make migrate
make seed           # loads curated career stubs

# 4. (Optional) Ingest O*NET data
# Download O*NET 28.x+ text files from https://www.onetcenter.org/database.html
# Extract into ./data/onet/, then:
make ingest-onet ONET_DIR=/data/onet

# 5. Start the frontend (separate terminal)
cd frontend
npm install
npm run dev
```

Open [http://localhost:3000](http://localhost:3000).

### Running tests

```bash
make test
```

Tests require the Docker backend to be running (`docker compose up -d`).

## Deployment

### Backend → Railway

1. Create a Railway project and add a Postgres plugin.
2. Point the service to this repo; set `RAILWAY_DOCKERFILE_PATH=backend/Dockerfile.production`.
3. Set all required environment variables in the Railway dashboard.
4. Deploy — migrations run automatically on startup (`alembic upgrade head`).

### Frontend → Vercel

1. Import the repository into Vercel; set root directory to `frontend/`.
2. Add `NEXT_PUBLIC_API_URL` pointing to your Railway backend URL.
3. Deploy.

After both are live, set `PUBLIC_BASE_URL` on the Railway service to your Vercel URL so CORS allows the production frontend.

## Limitations

- **US-centric career data** — O\*NET is a US government dataset. Job titles and skill expectations may differ in other countries.
- **No real-time labour market data** — salary ranges, demand trends, and hiring rates are not included.
- **LLM output is non-deterministic** — the same profile may produce different rankings across runs.
- **Resume parsing is heuristic** — accuracy depends on file quality and formatting. Scanned PDFs and heavily formatted documents often fail.
- **Course relevance scores are estimates** — the LLM assigns scores based on title, channel, and description snippet. It has not watched the content.
- **No accessibility audit** — the UI has not been formally audited against WCAG 2.1.
